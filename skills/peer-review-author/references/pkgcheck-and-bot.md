## pkgcheck

`pkgcheck` bundles `R CMD check`, `goodpractice`, `pkgstats`, plus ~25 rOpenSci
structural checks. Run it before submitting:

```r
pkgcheck::pkgcheck()
```

or use the GitHub Action `ropensci-review-tools/pkgcheck-action`. For unusual
system dependencies, submit a PR to the base Dockerfile in the pkgcheck repo.

### The checks pkgcheck runs

- Default GitHub branch is not `master`.
- All functions have examples.
- All functions document their return values.
- Has `inst/CITATION`.
- Has a contributing / `CONTRIBUTING` file.
- License is acceptable (CRAN/OSI).
- No failed lintr checks.
- `R/` is flat (no sub-directories).
- Number of `Imports` dependencies (flagged if high).
- No obsolete package dependencies.
- On CRAN or not (informational).
- Package name is available / its CRAN status.
- `renv` is de-activated if present.
- GitHub repo lists a website.
- Repo is not a fork.
- No junk files (e.g. `.DS_Store`).
- All function names are unique.
- Valid `URL` field in DESCRIPTION.
- Valid `BugReports` field in DESCRIPTION.
- No `\dontrun{}` in examples.
- Docs are generated by roxygen2.
- At least one vignette.
- Consistent left-assignment operators.
- `srr` standards documentation is complete (statistical packages only).
- All CI badge jobs currently pass.

Failures (❌) must be fixed; attention items (👀) need information or a fix before
the editor proceeds.

## @ropensci-review-bot — author-facing commands

The bot drives the review on the GitHub issue. Commands an author commonly uses:

- `@ropensci-review-bot check package` — re-run pkgcheck after pushing changes.
- (Editors/others run scope, reviewer assignment, review recording, and approval
  commands — see the bot cheatsheet.)

The full command set is at <https://devguide.ropensci.org/bot_cheatsheet.html>.
